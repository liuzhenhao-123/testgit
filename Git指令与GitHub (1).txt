先把代码写好，存在工作区

$ git add 'file' 添加指定文件/文件夹到暂存区。
$ git add . 添加所有文件/文件夹到暂存区。
$ git add -A  添加当前仓库中最新修改的文件到暂存区。
$ git status 查看当前状态。红色表示工作区，绿色表示暂存区。没有内容表示都已成功上传到历史区。
$ git rm --cached 'file' 将指定代码从暂存区撤回到工作区。

$ git commit 这是不带描述的，提交到历史区。
$ git commit -m'这是我第一次使用git。' 这是带描述的，提交到历史区。

$ git log 查看提交记录（历史版本信息）
$ git reflog 查看提交记录（包括回滚）


修改文件后重新上传到历史区。
$ git add -A
$ git status
$ git commit -m'我在第二天修改后的版本'
$ git log
$ git reflog
$ git reset --hard 0e93922 这是历史版本回退的功能。版本号可以只取前七位，也可以全部都黏贴。

GitHub是Open Source源代码管理平台。
熟知的插件类库框架都有在GitHub上托管。
控制面板--> 凭据管理器--> 储存密码
 
git remote -v 查看本地仓库和哪些远程仓库保持连接
git remote add origin https://github.com/liuzhenhao-123/git-upload.git 建立本地仓库与指定远程仓库的连接。Origin只是为连接起的一个名字。
git remote rm origin 断掉仓库间的连接。
git pull origin master 提交之前先拉取（可能有另外的管理者上传或直接在GitHub上修改上传了程序，此时再提交暂存区历史区）。
git push origin master 将本地代码提交到远程仓库。
弹出浏览器:输入账号和密码，确认提交。
或者取消掉浏览器窗口，在命令行输入GitHub的账户名以及密码，密码是暗文看不见，输完按enter键提交，代码开始Upload。

git branch -M main 创建新的branch分支
git push origin main 上传到GitHub仓库的main分支

实际项目开发流程：
1.组长在账号内创建中央仓库，添加协作者
2.组员git clone公共文件（免于init ，git remote add，git pull）
3.组员编写上传程序
4.组员pull其他组员的程序，git add . 提交暂存区，git commit -m''，提交历史区，git push到中央仓库。
